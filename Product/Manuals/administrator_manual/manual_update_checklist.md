# Manual Update Checklist

1. Identify affected sections (User vs Admin manual).
2. Update text/screenshots; add new page if scope warrants.
3. Append entry to `user_manual/release_notes.md` with sprint ID + summary.
4. Mention manual updates in sprint README (e.g., “Manual updated: Navigation, Dashboards”).
5. During sprint close, verify manual files changed (or explicitly note “No manual update required”).
